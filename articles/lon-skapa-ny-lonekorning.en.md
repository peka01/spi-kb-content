# Create a payroll run

Here you will learn how to create a new payroll run in **{{resource:Common_Company_Name_Short}}** **{{resource:Common_Product_Name_Variable_eAccCloudPayroll}}**. By initiating a payroll run, you can collect supporting documentation and manage payments for both regular salaries and extra payments in a structured way.

## Instruction

1. Go to the menu and select ****{{resource:CloudPayroll_EmployeeList_legend_Salary}}****, then click on ****{{resource:CloudPayroll_WageRuns}}****.
2. Click the button ****{{resource:CloudPayroll_WageRunList_hdr_CreateNewWageRun}}****.
3. Select the type of wage run in the drop-down list. Select ****{{resource:CloudPayroll_WageRunList_lbl_WagerunType_Ordinary}}**** to fetch calendar registrations and recurring lines, or ****{{resource:CloudPayroll_WageRunList_lbl_WagerunType_Extra}}**** to create empty payslips (for example, for bonuses).
4. Enter a ****{{resource:VoucherType}}**** for the wage run if you want to make it easier to identify in the list.
5. Check ****{{resource:CloudPayroll_QuickWagerun_PaymentDate_lbl}}****. The system suggests a date based on previous runs, but you can change this manually.
6. Check the box ****{{resource:CloudPayroll_WageRunForm_btn_Import_Data}}**** if you want to fetch data from features such as ****{{resource:Common_Product_Name_variable_time_project_planning}}**** or ****{{resource:Common_Product_Name_variable_travel_expense}}****.
7. Use the filters for ****{{resource:CloudPayroll_EmployeeList_lbl_Basic_Pay_Type}}**** or ****{{resource:CloudPayroll_EmployeeList_lbl_Workforce_Category}}**** if you want to limit which employees are displayed in the list.
8. Check that all employees to be included in the run are selected in the list on the left.
9. Click the button ****{{resource:CloudPayroll_QuickWagerun_CreateQuickWageRun_btn}}****.

## Tips
> For an employee to appear in the list, all mandatory fields in the employment register must be filled in. If a person is missing, check that personal details and employment dates are correctly registered.

> If the selected payment date falls on a public holiday, a warning is displayed. Make sure to select the nearest business day and send the payment file in good time to ensure that the salary reaches the employees on time.
